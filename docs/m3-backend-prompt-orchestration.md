# M3.1 后端 Prompt 编排

## 本阶段目标

这一阶段把新闻助手页面里与提示词相关的逻辑全部收回后端，前端只负责：

- 采集用户问题
- 传递有限历史消息
- 传递回答模式、时间范围、关注主题
- 展示模型回复和后端返回的策略信息

这样做的目的有三点：

1. 避免系统提示词和约束直接暴露在前端。
2. 为后续接入 RAG、LangGraph、Agent 工作流保留稳定入口。
3. 让提示词版本、策略和历史裁剪逻辑可以统一在后端维护。

## 本次改动

### 1. 新增后端 Prompt 模块

新增 `backend/prompts/news_assistant.py`，集中维护：

- `PROMPT_VERSION`
- 系统提示词
- 回答模式配置
- 时间范围配置
- 主题配置
- 历史消息裁剪逻辑
- 最终消息拼装逻辑

当前版本号为 `news-assistant-v1`。

### 2. AI 接口改为结构化入参

`/api/ai/chat` 现在接收的请求体不再是前端拼好的 `messages`，而是：

```json
{
  "question": "帮我总结最近科技热点",
  "history": [
    { "role": "user", "content": "先说重点" },
    { "role": "assistant", "content": "好的" }
  ],
  "mode": "brief",
  "timeRange": "7d",
  "category": "technology"
}
```

后端会根据这些字段统一构建最终发送给模型的消息列表。

### 3. 历史消息裁剪

为了降低上下文膨胀，后端会对历史消息做两层限制：

- 最多保留最近 8 条历史消息
- 总字符数限制在约 1500 字以内

这一步是后续做会话摘要和 Agent 状态管理的基础。

### 4. 前端改成纯参数传递

AI 页不再维护系统提示词，也不再拼接“模式要求”“主题要求”这类用户提示。

前端现在只会把：

- 原始问题
- 现有会话历史
- mode
- timeRange
- category

发到后端。

## 当前边界

这一阶段仍然不是完整的新闻 Agent：

- 还没有接入新闻库检索
- 还没有 RAG
- 还没有 LangGraph 工作流
- 还没有引用来源和置信度

但它已经把 AI 能力的控制面从前端迁到了后端，这是后续继续升级的必要前提。

## 手动测试

1. 启动后端和前端。
2. 打开新闻助手页，任选模式、时间范围和主题。
3. 连续发两到三轮消息，确认页面仍能正常对话。
4. 观察页面顶部的 `Prompt` 版本和 `策略` 标签，确认已显示后端返回的信息。
5. 切换不同模式后发送相似问题，观察回答结构是否仍有明显变化。
6. 点击“新对话”，确认会话被清空并恢复欢迎消息。
7. 如果要直接验证后端提示词拼装，可以在后端用 `prepare_chat_messages()` 检查最终消息列表。
