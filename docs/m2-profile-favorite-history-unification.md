# M2.3：个人中心、收藏页、历史页统一升级

## 目标

这一阶段的重点，是把“我的 / 收藏 / 历史”从三组彼此割裂的页面，整理成一套统一的移动端产品体验。

主要目标：

- 个人中心成为真正的导航中枢，而不是简单菜单页
- 收藏页和历史页共享统一的新闻卡片样式
- AI 页入口更自然地出现在个人中心链路中
- 保持现有后端接口不变，只优化前端组织与体验

## 本次改动

### 1. 新增共享新闻卡片组件

新增：

- `frontend/src/components/SavedNewsCard.vue`

用途：

- 统一渲染收藏页和历史页中的新闻条目
- 统一处理无图占位、发布时间、阅读量、附加时间字段

这样做的价值是：

- 收藏页和历史页不再各写一套相似模板
- 后续如果要做“我的推荐”“最近问答引用来源”，也可以继续复用

### 2. 重构个人中心页

重写：

- `frontend/src/views/My.vue`

新页面结构：

- 顶部个人资料 hero 区
- 收藏数 / 历史数 / AI 助手 三个摘要卡
- 服务入口列表

改完后的个人中心承担了更明确的产品职责：

- 展示账号状态
- 汇总个人阅读沉淀
- 为 AI 页提供自然入口

### 3. 重构收藏页

重写：

- `frontend/src/views/Favorite.vue`

优化点：

- 顶部收藏概览卡
- 收藏列表统一卡片化
- 清空、删除操作收口
- 未登录时给出登录入口

### 4. 重构历史页

重写：

- `frontend/src/views/History.vue`

优化点：

- 顶部阅读轨迹概览卡
- 列表样式与收藏页统一
- 删除、清空逻辑更清晰
- 保留 API 优先、本地回退的加载方式

### 5. 优化底部导航栏

更新：

- `frontend/src/components/TabBar.vue`

变化：

- 去掉多余的本地 active 状态维护
- 统一底栏阴影和激活色

## 为什么这一步重要

### 1. 让项目更像完整产品，而不是页面集合

很多项目首页和详情页做得还可以，但个人中心链路明显像“补出来的”。

这一步的价值，就是把用户相关页面也拉到同一设计水位。

### 2. 让“新闻消费链路”更完整

现在链路已经更清楚：

- 首页：发现新闻
- 详情页：阅读新闻
- 收藏/历史：沉淀新闻
- AI 页：分析新闻

这比单纯堆功能更像真实业务产品。

### 3. 为后续 Agent 场景预留位置

个人中心里现在已经有 AI 助手入口，后续接新闻 Agent 后，用户对“问答”和“阅读”的切换会更自然。

## 手工测试建议

### 1. 验证个人中心摘要卡

1. 打开“我的”页面
2. 观察是否出现收藏数、历史数、AI 助手三个摘要卡

预期：

- 页面有更完整的个人中心结构
- 点击收藏数和历史数能跳到对应页面
- 点击 AI 助手能跳到 AI 页

### 2. 验证收藏页

1. 进入收藏页
2. 检查卡片样式是否统一
3. 点击任一收藏项，确认能进入详情页
4. 删除单条收藏，再检查列表是否更新
5. 点击“清空”，确认所有收藏被清空

### 3. 验证历史页

1. 进入历史页
2. 检查卡片样式是否与收藏页一致
3. 点击任一历史项，确认能进入详情页
4. 删除单条历史，再检查列表是否更新
5. 点击“清空”，确认所有历史被清空

### 4. 验证未登录场景

1. 退出登录
2. 进入“我的”页面
3. 观察是否能看到登录/注册按钮
4. 进入收藏页或历史页

预期：

- 页面不会崩
- 如果没有本地数据，会给出空态和登录入口

## 面试可讲的点

- 为什么要把个人中心做成阅读沉淀页，而不只是菜单页
- 为什么收藏页和历史页要抽共享卡片组件
- 为什么 AI 助手入口要放进个人中心链路
- 如何在不改后端接口的前提下提升前端产品完成度
